# プロジェクト構造ルール

## スキルとエージェントの連携パターン

- **スキル（skills/）**: ユーザー向けのエントリポイント。ワークフローを制御する
- **エージェント（agents/）**: スキルから `Task()` で委譲される実行単位
- スキルがオーケストレーション、エージェントが実処理を担当する

## テンプレート・フォーマットの配置ルール

- テンプレートやフォーマット定義は **エージェント側の `references/`** に配置する
- スキル側にはテンプレートやフォーマットファイルを置かない
- スキルからエージェントにパスを渡す際は、エージェントの `references/` パスを指定する

## feature-state.json のフェーズ遷移

```
spec → build → check → done
```

- phase フィールドは現在アクティブなフェーズを示す
- 各スキルが開始時・完了時に phase を更新する（feature-state-manager は廃止）
- check の NEEDS_FIX 時は phase を "spec" に戻す（ループ対応）
- 各フェーズの status は pending / in_progress / done の3値

## 評価駆動開発

- スキル作成前に評価シナリオ（入力→期待出力）を先に作る
- ベースライン測定 → 最小限のドキュメント作成 → 再測定 のサイクルで改善する
- 評価に合格してからスキルを公開する

## MCP ツール参照

- MCP ツールは完全修飾名 `ServerName:tool_name` で参照する
- `allowed-tools` にも同じ形式で記述すること
